Практическая работа №15

Организации памяти в STM32. Flash-память STM32. Процесс начальной загрузки Cortex-M



Контрольные вопросы


1. Что происходит когда компилируется микропрограмма, используя инструментарий GCC ARM?


При компиляции микропрограммы с помощью инструментария GCC ARM происходит преобразование исходного кода на языке С в машинный код, который может быть выполнен на STM32.


2. Как выглядит типовая организации Flash-памяти в микроконтроллере STM32?


Типовая организация Flash-памяти в микроконтроллере STM32 состоит из блоков фиксированного размера, которые могут быть стерты или перезаписаны отдельно.


3. Как организована память SRAM в микроконтроллере STM32?


Память SRAM в микроконтроллере STM32 организована в блоках переменного размера, которые используются для временного хранения данных.


4. Какие программные структуры содержат файлы с исходным кодом Си?


Файлы с исходным кодом Си содержат программные структуры, такие как функции, переменные и определения типов.


5. Что такое объектный файл и какой из стандартов использует GCC ARM?


Объектный файл - это результата компиляции, который содержит машинный код и информацию о символах и реализации функций. GCC ARM использует формат ELF (Executable and Linkable Format).


6. В чём особенность памяти, связанной с ядром CCM?


Память CCM (Core Coupled Memory) - это быстрая и небольшая SRAM, которая расположена рядом с ядром процессора. Она используется для хранения критически важных данных, которые должны обслуживаться без задержек.


7. Как может быть организована Flash-память микроконтроллера STM32 в зависимости от семейства STM32, вида поставки и используемого корпуса?


Организация Flash-памяти микроконтроллера STM32 может отличаться в зависимости от семейства, вида поставки и корпуса. Она может быть разбита на блоки разных размеров, иметь дополнительную защиту данных, различные временные характеристики и т.д.


8. Суть памяти NAND-Flash


NAND-Flash - это форма энергонезависимой памяти, обычно используемая для хранения больших объемов данных в мобильных устройствах и платах.


9. В чём особенности устройства NOR-Flash памяти?


NOR-Flash - это более быстрая форма энергонезависимой памяти, которая используется для хранения кода встраиваемых систем.


10. Как избежать нежелательных записей в энергонезависимую память (NVM)?


Чтобы избежать нежелательных записей в NVM, следует использовать механизм защиты записи и выбирать правильный тип NVM для конкретного приложения.


11. Что такое байты конфигурации?


Байты конфигурации - это определенные байты в энергонезависимой памяти, которые используются для хранения информации о конфигурации микроконтроллера.


12. Наиболее распространенные параметры конфигурации


Наиболее распространенные параметры конфигурации - это тактовые частоты, режимы работы и настройки входов/выходов.


13. Что представляет собой однократно программируемая (OTP) память?


Однократно программируемая (OTP) память - это форма энергонезависимой памяти, которая может быть программирована только один раз.


14. В чём суть механизма физического перераспределения памяти?


Механизм физического перераспределения памяти используется для устранения проблем с перегрузкой и разрядом памяти.


15. Какую функцию выполняет загрузчик?


Загрузчик выполняет процесс начальной загрузки Cortex-M, проверяет наличие и работоспособность программного обеспечения, загружает и запускает приложение.


16. Какие существуют ограничения в отношении управления памятью через загрузчик?


Ограничения в отношении управления памятью через загрузчик могут быть связаны с ограниченным объемом памяти и доступом к определенным ресурсам системы.
